# TXT Nova Toolkit 开发计划

## 目标

开发一个 Calibre GUI 插件 `TXT Nova Toolkit`，提供独立的 TXT 小说导入、更新、导出入口，用插件逻辑接管文件名解析、连载更新、标签设置、旧格式清理和按简介导出，不依赖 Calibre 自带 Auto Merge。

插件核心目标：

- 普通新增书籍时，按固定文件名解析元数据。
- 连载书籍再次导入时，自动更新已有书籍记录。
- 完结、完本、全本书籍不自动合并，避免误覆盖历史记录。
- 更新连载书籍时，以新导入 TXT 为基准，清理旧派生格式。

## Calibre 推荐设置

Calibre 自带 Auto Merge 建议关闭：

```text
Preferences -> Import/export -> Adding books -> Adding actions
Auto-merge added books if they already exist: off
```

原因：Calibre 自带 Auto Merge 不能判断“连载 / 完结”，也不能自动删除旧派生格式。插件应提供自己的导入按钮，避免 Calibre 默认导入流程先行合并。

Calibre 文件名元数据正则可以保留，用于普通手动添加书籍，但插件导入流程不依赖它。

## 插件入口

插件类型：`Interface Action Plugin`

GUI 中新增按钮：

```text
导入/更新 TXT 小说
```

插件按钮带下拉菜单：

```text
导入/更新 TXT 小说
导出选中小说
```

按钮行为：

1. 选择一个或多个 `.txt` 文件。
2. 对每个文件解析标准文件名。
3. 判断状态和标签。
4. 根据规则新增或更新 Calibre 书籍记录。
5. 最后弹出处理结果汇总。

## 文件名格式

来源工具生成的标准文件名格式：

```text
《书名》 起始-结束章 状态 作者：作者.txt
```

示例：

```text
《三体》 1-120章 连载 作者：刘慈欣.txt
《三体》 1-300章 L连载 作者：刘慈欣.txt
《三体》 1-300章 完结 作者：刘慈欣.txt
```

解析字段：

- `title`：书名号内文本，不包含 `《》`。
- `author`：`作者：` 后面的文本。
- `comments`：文件名主体，不包含 `.txt` 扩展名。
- `status_text`：书名和作者之间的中间文本，用于识别 `连载`、`完结`、`断更`、`NTR`、`L` 等标志。

## 元数据写入规则

新增书籍时写入：

- `title`：解析出的书名。
- `authors`：解析出的作者。
- `comments`：文件名主体，不包含 `.txt` 扩展名。
- `series`：默认 `未导入`；如果导入 TXT 小于 `500 KB`，设置为 `太短了，先养一养`。
- `tags`：按标签规则生成的组合标签。
- `timestamp/date added`：当前导入时间。
- `pubdate/出版日期`：固定设置为 `2000-12-31`，用于区分插件新导入/更新过的书籍。
- `publisher/出版方`：默认设置为 `中国成人文学精选`。

更新已有连载书时写入：

- 保留已有书籍记录。
- 保留 `title` 和 `authors`，原则上它们应与新导入文件一致。
- 更新 `comments`：将新导入文件名主体插入到原简介最上方，不包含 `.txt` 扩展名。
- `comments` 最多保留 5 行；如果插入新行后超过 5 行，删除最下面的旧行。
- 保留原有 `tags`；如果原本没有任何标签，则按新导入文件名生成组合标签。
- 更新 `timestamp/date added` 为当前导入时间。
- 更新 `pubdate/出版日期` 为固定日期 `2000-12-31`。
- 如果原 `publisher/出版方` 为空，则设置为 `中国成人文学精选`；已有出版方不覆盖。
- 更新 `last_modified` 由 Calibre 元数据/格式变更自然触发；如 API 支持，也主动刷新。
- 保留原有封面、评分、自定义列和其他非简介元数据。
- 如果导入 TXT 小于 `500 KB`，将 `series` 设置为 `太短了，先养一养`。

## 合并与新增规则

状态判断优先级：

1. 如果文件名包含 `完结`、`完本`、`全本`，强制视为完结，不自动合并。
2. 如果文件名包含 `连载`，视为连载，允许自动查找并更新已有记录。
3. 如果文件名包含 `断更`，视为断更，默认不自动合并。
4. 其他情况作为普通新增书籍处理。

处理规则：

| 文件类型 | 是否查找已有记录 | 找到已有记录时 | 找不到时 |
|---|---:|---|---|
| 连载 | 是 | 更新已有记录 | 新增书籍 |
| 完结 / 完本 / 全本 | 是 | 新增重复记录并标记 `重复` | 新增书籍 |
| 断更 | 是 | 新增重复记录并标记 `重复` | 新增书籍 |
| 其他 | 否 | 不合并 | 新增书籍 |

说明：完结、完本、全本、断更书籍即使匹配到同书名同作者，也不覆盖已有记录，而是创建一条新记录，并将新记录标签设置为 `重复`，交由用户人工判断。

## 连载更新规则

当新导入 TXT 被识别为连载，且 Calibre 中存在同书名、同作者记录时：

1. 将新 TXT 加入已有书籍记录。
2. 如果已有记录已有 TXT 格式，用新 TXT 覆盖旧 TXT。
3. 删除该书籍记录中除 TXT 以外的所有格式，例如：
   - `EPUB`
   - `AZW3`
   - `MOBI`
   - `PDF`
   - 其他非 TXT 格式
4. 更新简介：新文件名主体作为第一行，原简介内容下移。
5. 更新时间为当前导入日期。
6. 保留原有标签；如果原本没有任何标签，则按新导入文件名生成组合标签。
7. 保留原有封面、评分、自定义列和其他非简介元数据。

这样可以保证这本书以新 TXT 为唯一基准，旧版 TXT 派生出的电子书格式不会残留。

## 简介历史规则

更新已有连载书籍时，简介用于记录最近导入历史：

1. 本次导入的文件名主体写在第一行。
2. 原简介内容保留并整体下移。
3. 最多保留 5 行。
4. 如果超过 5 行，删除最下面的一行。
5. 空行不计入历史。

## 重复记录标记规则

当新导入 TXT 被识别为完结、完本、全本或断更，且 Calibre 中存在同书名、同作者记录时：

1. 不更新已有记录。
2. 新增一条独立书籍记录。
3. 新记录标签设置为 `重复`。
4. 新记录简介写入文件名主体，不包含 `.txt` 扩展名。
5. 用户后续自行判断是否保留、合并或删除。

该规则用于避免完结版或断更版误覆盖已有连载记录。

## 已有记录匹配规则

匹配条件：

```text
title 完全一致
author 完全一致
```

建议初版不做模糊匹配，避免误合并。

后续可选增强：

- 忽略书名首尾空格。
- 忽略作者首尾空格。
- 支持中文全角/半角冒号统一。
- 支持手动选择多个候选记录。

## 标签规则

新增普通书籍时，标签写入为一个组合标签字符串，保持与现有 Calibre 标签一致，例如 `刘备、连载、加料`。

更新已有连载书籍时，不按文件名重写已有标签；保留原标签。如果原记录没有任何标签，则按新导入文件名生成组合标签。

完结、完本、全本、断更书籍如果命中已有同名同作者记录，则新增记录的标签设置为 `重复`。

基础规则：

- 默认大类为 `刘备`。
- `NTR` 优先于默认 `刘备`。
- `L` 表示 `加料`。
- 状态归一化为 `连载`、`完结`、`断更`。

组合关系：

| 文件名标志 | 写入标签 |
|---|---|
| `NTR连载` | `NTR、连载` |
| `NTR完结` | `NTR、完结` |
| `NTR断更` | `NTR、断更` |
| `L连载` | `刘备、连载、加料` |
| `L完结` | `刘备、完结、加料` |
| `L断更` | `刘备、断更、加料` |
| `连载` | `刘备、连载` |
| `完结` / `完本` / `全本` | `刘备、完结` |
| `断更` | `刘备、断更` |
| 只有 `L` 或 `加料` | `加料` |
| `刺猬猫` + `完结` | `刺猬猫、完结` |
| `起点` + `完结` | `起点、完结` |
| `调教` | `调教` |
| `母女` | `母女` |
| `母子` | `母子` |
| `文学` | `文学` |
| `自购` | `自购` |

初版重点实现：

- `NTR连载 / NTR完结 / NTR断更`
- `L连载 / L完结 / L断更`
- `连载 / 完结 / 完本 / 全本 / 断更`

其他标签可作为后续增强，或初版用关键词表实现。

## 日期规则

新增书籍：

- 使用当前导入时间作为 Calibre 日期。

更新已有连载书：

- 将 Calibre 书籍记录的日期更新为当前导入时间。
- 目的是让按日期排序时，刚更新的连载书排到前面。

出版日期规则：

- 插件新增或更新的书籍，统一将 `pubdate/出版日期` 设置为 `2000-12-31`。
- 该固定日期用于快速筛选和区分插件处理过的导入记录。

出版方规则：

- 插件新增的书籍，统一将 `publisher/出版方` 设置为 `中国成人文学精选`。
- 插件更新已有连载书籍时，如果原出版方为空，则补为 `中国成人文学精选`。
- 如果原出版方已有值，则保持原值，不覆盖。

## 小文件丛书标记规则

当导入 TXT 文件大小小于 `500 KB` 时：

```text
series = 太短了，先养一养
```

该规则用于标记内容体量太小、暂时不值得继续处理的连载文件。

适用范围：

- 新增书籍。
- 更新已有连载书籍。
- 新增重复记录。

如果文件大小大于等于 `500 KB`，则不因为该规则改写已有记录的丛书字段；新增书籍仍按默认规则设置为 `未导入`。

## 异常处理

文件名无法解析：

- 不导入该文件。
- 在结果汇总中列为失败，并显示原因。

连载书匹配到多个已有记录：

- 初版建议不自动更新。
- 在结果汇总中标为需要人工处理。
- 后续可加弹窗让用户选择目标记录。

添加或删除格式失败：

- 保留错误信息。
- 不删除源文件。
- 在结果汇总中显示失败项。

## 不做的事情

初版不做：

- 不修改 Calibre 全局 Auto Merge 设置。
- 不监听 Calibre 默认“添加书籍”按钮。
- 不做模糊匹配合并。
- 不自动转换 EPUB/AZW3/MOBI。
- 不删除用户磁盘上的原始 TXT 文件。

## 导出规则

导出入口：

```text
导出选中小说
```

处理流程：

1. 用户在 Calibre 书库中选中一本或多本书。
2. 插件要求用户选择导出目录。
3. 插件弹出 Calibre 风格的格式选择窗口，直接选中格式；按 Ctrl/Shift 可多选。
4. 对每本书读取 `comments/简介` 的第一行作为导出文件名主体。
5. 导出文件自动补所选格式对应扩展名，例如 `.txt`、`.epub`、`.azw3`。
6. 从书籍记录中复制所选格式文件到目标目录。
7. 如果简介为空，则回退到 `书名 - 作者`。
8. 如果目标目录存在同名文件，自动追加序号，例如 `书名 (2).txt`。
9. 如果书籍没有所选格式，则跳过并在结果汇总中显示失败原因。
10. 导出完成后自动打开目标目录。

文件名安全规则：

- 替换 Windows 文件名非法字符 `\\ / : * ? " < > |` 为 `_`。
- 去掉首尾空格和末尾点号。

## 开发步骤

1. 创建 Calibre 插件骨架。
2. 添加 GUI 按钮和多文件选择入口。
3. 实现标准文件名解析函数。
4. 实现标签识别函数。
5. 实现新增书籍流程。
6. 实现连载书查找已有记录逻辑。
7. 实现连载更新逻辑：覆盖 TXT、删除非 TXT 格式、更新简介、标签、日期。
8. 实现结果汇总弹窗。
9. 打包为可安装 `.zip` 插件。
10. 准备测试用例和手动验证清单。

## 测试样例

新增连载：

```text
《测试书》 1-100章 连载 作者：测试作者.txt
```

更新连载：

```text
《测试书》 1-200章 连载 作者：测试作者.txt
```

加料连载：

```text
《测试书》 1-200章 L连载 作者：测试作者.txt
```

NTR 连载：

```text
《测试书》 1-200章 NTR连载 作者：测试作者.txt
```

完结不合并：

```text
《测试书》 1-300章 完结 作者：测试作者.txt
```

预期结果：

- 连载再次导入会更新已有记录。
- 完结同名书不会自动合并。
- 更新连载后只保留 TXT 格式。
- 更新连载后简介变成最新文件名。
- 更新连载后日期变为当天。
- 标签符合组合规则。

## 待确认点

1. `简介` 是否必须包含 `.txt` 扩展名？当前计划是包含。
2. 完结书同名导入时是否允许 Calibre 创建重复记录？当前计划是允许。
3. 断更书是否永远不合并？当前计划是不合并。
4. 更新已有连载时，保留原来的封面、评分、自定义列和其他非简介元数据，只清除并替换简介。
5. 更新已有连载时，保留原有标签；如果原本没有标签，则按新导入文件名生成组合标签。
6. 导入 TXT 小于 `500 KB` 时，丛书设置为 `太短了，先养一养`。
